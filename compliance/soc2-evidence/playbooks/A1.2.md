# Playbook — A1.2 Backup and recovery

- **Control ID:** A1.2
- **Control description:** The entity authorizes, designs, develops or acquires, implements, operates, approves, maintains, and monitors environmental protections, software, data backup processes, and recovery infrastructure to meet its objectives.
- **Evidence-collection mechanism:** Daily hash of `wixie/plugins/inference-engine/state/artifacts.jsonl` (append-only master log) plus each repo's `state/precedent-log.md`. Git history provides the durable backup; each repo's GitHub mirror is the off-host copy.
- **Producer (today):** `wixie/plugins/inference-engine/`, `*/state/precedent-log.md`, git/GitHub.
- **Retention rule:** 7 years.
- **Who reviews:** Repo maintainer weekly.
- **Escalation path:** Append-only artifacts.jsonl rewritten (hash regresses) → corruption; investigate per inference-substrate.md anti-pattern guidance.
- **Type II evidence shape:** Continuous append-only growth of artifacts.jsonl + committed precedent-logs over the window; git mirror confirms off-host redundancy.
- **External dependency:** GitHub (the git mirror).
