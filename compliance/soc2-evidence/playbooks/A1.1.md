# Playbook — A1.1 Capacity for processing demand

- **Control ID:** A1.1
- **Control description:** The entity maintains, monitors, and evaluates current processing capacity and use of system components (infrastructure, data, and software) to manage capacity demand and to enable the implementation of additional capacity to help meet its objectives.
- **Evidence-collection mechanism:** Daily presence-check + state listing for `pech/plugins/budget-watcher/`, `pech/plugins/rate-shield/`, `emu/plugins/context-guard/`.
- **Producer (today):** `pech/plugins/budget-watcher/`, `pech/plugins/rate-shield/`, `emu/plugins/context-guard/`.
- **Retention rule:** 1 year.
- **Who reviews:** Repo maintainer weekly.
- **Escalation path:** State dir missing or empty for >7 days → plugin not running. **GAP:** F-013 multi-tenant rate-limiting absent — must close before SaaS Type II.
- **Type II evidence shape:** Continuous metric streams from the three plugins across the window.
- **External dependency:** None for developer-local; future SaaS deployment will require infrastructure capacity metrics from the hosting provider.
